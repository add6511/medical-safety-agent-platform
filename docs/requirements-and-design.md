# 基层医疗安全型预问诊与随访平台 — 需求规格与系统总体设计

> **安全声明：本平台仅供教学演示，不提供真实医疗诊断、处方或治疗建议。所有病例数据均为合成教学数据，不包含任何真实患者信息。**

---

## 1. 项目背景与目标

### 1.1 项目背景

基层医疗机构在接诊过程中面临分诊效率低、高风险症状识别不及时等问题。为辅助基层医务人员进行预问诊和分诊决策，本项目构建了一个**安全型预问诊与随访平台**的教学演示原型。

本平台的核心设计原则是**安全优先**：通过确定性规则引擎 + 多 Agent 协作审核 + 输入/输出安全检测的多层防护机制，确保高风险症状不被错误降级，不安全内容不被返回给用户。

### 1.2 项目定位

- **教学演示平台**：用于展示医疗 AI 辅助筛查的安全设计思路和工程实践
- **仅供教学演示**：不提供真实诊断、处方或治疗建议
- **只使用合成演示数据**：所有病例、知识库内容、评测数据集均为合成生成，不包含真实患者信息
- **不可直接用于临床**：评测结果不得宣传为真实临床准确率

### 1.3 项目目标

1. 演示基于多 Agent 协作的预问诊安全审核流程
2. 展示确定性规则引擎对 AI 模型风险评估的兜底保护机制
3. 实现输入安全检测（提示词攻击、PII 脱敏）和输出安全检测（确定性诊断拦截、药物处方拦截）
4. 提供合成病例评测与消融对比实验的完整工具链
5. 探索 LoRA 微调在小规模医疗安全场景中的应用（V1/V2 已完成）

## 4. 功能需求

### 4.1 当前已实现功能

#### 4.1.1 知识库文档管理

提供教学文档的完整生命周期管理能力。

- **文档导入** (`POST /api/v1/knowledge/documents`)
  - 支持导入教学用指南文本，包含标题、发布机构、来源链接、版本号、发布时间、正文内容
  - 自动计算 SHA-256 校验值，相同校验值的文档幂等导入，不重复创建数据
  - 文档内容限制最大 500,000 字符，仅接受合成教学材料
  - 写接口支持通过 `X-Internal-API-Key` 请求头进行内部鉴权

- **文档列表查询** (`GET /api/v1/knowledge/documents`)
  - 返回已导入文档列表，包含元信息但不包含正文

- **文档删除** (`DELETE /api/v1/knowledge/documents/{document_id}`)
  - 删除指定文档及其关联的所有知识块

- **知识检索** (`POST /api/v1/knowledge/search`)
  - 接收查询文本，返回最相关的知识片段（1-10 条）
  - 检索结果包含相似度得分、文档来源、发布机构等完整引用信息

#### 4.1.2 RAG 知识检索

实现完整的检索增强生成 (RAG) 基础设施：

- **文档切分**：基于 langchain-text-splitters 的 `RecursiveCharacterTextSplitter`，针对中文医学文本配置合理的分隔符
- **向量化**：
  - `mock` 模式：使用 SHA-256 哈希生成确定性归一化向量（384 维），无需外部模型
  - `openai_compatible` 模式：通过 HTTPX 调用 OpenAI 兼容的 `/embeddings` 接口
- **向量存储**：
  - `memory` 模式：基于内存的向量存储，支持余弦相似度检索
  - `pgvector` 模式：基于 PostgreSQL + pgvector 扩展的持久化存储
- **优雅降级**：知识库为空或不可用时，检索环节自动降级，不中断整体审核流程

#### 4.1.3 合成安全规则引擎

确定性规则引擎，独立于 AI 模型进行风险评估：

- **规则数据源**：从 `synthetic_rules_v1.json` 加载合成规则集
- **匹配逻辑**：支持红旗标识匹配和关键词匹配，按优先级降序匹配
- **风险等级**：四级风险体系 — LOW、MEDIUM、HIGH、CRITICAL
- **不可下调机制**：HIGH/CRITICAL 一旦由规则引擎命中，不允许语言模型将风险降低
- **强制人工审核**：HIGH 和 CRITICAL 风险等级必须标记 `needs_human_review = true`
- **版本追踪**：规则集带版本号，支持版本追溯

#### 4.1.4 多 Agent 安全审核流程

采用 SupervisorAgent 协调四个专用 Agent 的流水线架构：

```
IntakeAgent → RetrievalAgent → RiskAssessmentAgent → SafetyReviewAgent
```

1. **IntakeAgent（输入校验与规范化）**
   - 规范化症状输入（名称、严重程度 0-10、持续时间）
   - 识别缺失字段（年龄、症状、自由文本）
   - 生成追问问题
   - 过滤无效红旗标识
   - 生成症状摘要文本

2. **RetrievalAgent（知识检索）**
   - 基于症状和自由文本构建查询
   - 调用 KnowledgeService 进行语义检索
   - 知识库为空时优雅降级

3. **RiskAssessmentAgent（风险评估）**
   - 执行确定性规则引擎评估
   - 解析模型建议的风险等级
   - 实施风险不可下调机制
   - LOW/MEDIUM 场景下取规则和模型中的较高等级

4. **SafetyReviewAgent（安全审核）**
   - 检查输出是否包含确定性诊断表述
   - 检查是否包含药物处方或具体剂量
   - 检查是否包含取消人工审核的危险表述
   - 检查高风险是否标记人工审核
   - 对不安全内容进行拦截替换

**审计记录**：每个 Agent 执行后自动生成审计记录，包含执行状态、输入摘要、输出摘要、耗时、关联规则 ID 等，不含思维链或内部推理过程。

#### 4.1.5 预问诊审核接口

`POST /api/v1/preconsultation/review`

- 接收合成教学病例信息（case_id、age、symptoms、red_flags、free_text、model_suggested_risk）
- 执行完整的多 Agent 安全审核流程
- 在流程前对所有用户输入字段进行统一的输入安全检测
- 对 free_text 单独进行 PII 脱敏
- 返回完整的审核结果，包括风险等级、规则命中、知识引用、安全标记、Agent 审计记录、版本信息

#### 4.1.6 分诊分析接口

`POST /api/v1/triage/analyze`

- 接口结构与预问诊审核类似
- 内部复用 PreconsultationService 的多 Agent 审核流程
- 返回统一输出结构，包含风险等级、症状摘要、红旗标识、证据列表、引用来源、缺失信息、追问问题、安全状态

#### 4.1.7 安全检查接口

`POST /api/v1/safety/check`

- 独立的安全检查接口，不执行完整多 Agent 流程
- 集成输入安全检测（提示词攻击 + PII）和输出安全检测（确定性诊断、药物处方、取消审核）
- 支持风险降级检查（HIGH/CRITICAL 时强制人工审核）
- 返回安全状态（pass / blocked / human_review）和清理后的文本

#### 4.1.8 输入安全检测

基于规则的教学安全检测模块（`input_guard.py`）：

- **提示词攻击检测**：中文/英文提示词注入、系统提示词提取尝试、越权角色指令、人工审核绕过尝试
- **PII 检测与脱敏**：中国大陆手机号、身份证号码、电子邮箱、银行卡号自动脱敏
- **安全状态判定**：检测到提示词攻击时标记为 `is_blocked`，PII 检测时标记 `needs_human_review`

#### 4.1.9 输出安全检测

SafetyReviewAgent 和 safety/check 接口中的输出安全检测：

- **确定性诊断拦截**：检测"你就是/得了/患有/确诊为"等表述模式
- **药物处方拦截**：检测"服用/口服/注射 XX mg/ml/片/粒"等剂量模式
- **取消审核拦截**：检测"不需要/无需人工审核"等危险表述
- **拦截后处理**：替换为安全说明文本，标记 `safety_status = blocked`

#### 4.1.10 合成病例评测与消融对比

完整的评测框架，用于验证多 Agent 安全审核流程的有效性：

- **评测数据集**：`synthetic_cases_v1.json`，包含 51 个合成教学案例
- **评测指标**：风险分级一致率、高风险召回率、RAG 引用命中率、结构化 JSON 有效率、安全拦截成功率、平均/P50/P95 响应延迟
- **三方案消融对比**：`no_rag`、`rag_only`、`rag_multi_agent`
- **报告输出**：自动生成 JSON 汇总、CSV 逐案例结果、Markdown 评测报告

## 7. 核心业务状态说明

### 7.1 预问诊审核流程状态

```
INITIATED
    |
    v
SYMPTOM_COLLECTED
    |
    v
AI_TRIAGE_COMPLETED ------------------+
    |                                  |
    +----- MEDICAL_REVIEW_COMPLETED    |
    |           |                      |
    |           v                      |
    |       COMPLETED (终态)            |
    |                                  |
    +----- NEEDS_REVISION              |
                |                      |
                v                      |
        AI_TRIAGE_COMPLETED ----------+
                                     (循环)
    任意状态 --> CANCELLED (终态)
```

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| INITIATED | 已创建 | 患者提交预问诊 |
| SYMPTOM_COLLECTED | 症状已收集 | 症状录入完成 |
| AI_TRIAGE_COMPLETED | AI 分诊完成 | 多 Agent 审核流程完成 |
| MEDICAL_REVIEW_COMPLETED | 医务审核完成 | 医务人员确认审核结果 |
| NEEDS_REVISION | 需要修订 | 医务人员驳回或要求补充信息 |
| COMPLETED | 已完成（终态） | 审核通过并归档 |
| CANCELLED | 已取消（终态） | 患者或管理员取消 |

> **说明**：COMPLETED 和 CANCELLED 为终态，不可继续转换。NEEDS_REVISION 回到 AI_TRIAGE_COMPLETED 重新执行 AI 审核。

### 7.2 安全状态

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| pass | 安全检查通过 | 未检测到安全问题 |
| human_review | 需人工审核 | HIGH/CRITICAL 风险或 PII 检测 |
| blocked | 已拦截 | 检测到提示词攻击、确定性诊断、药物处方或取消审核表述 |

### 7.3 风险等级体系

| 等级 | 严重程度值 | 说明 | 是否需要人工审核 |
|------|:---:|------|:---:|
| LOW | 0 | 低风险，常规处理 | 是（常规审核） |
| MEDIUM | 1 | 中等风险，需关注 | 是（常规审核） |
| HIGH | 2 | 高风险，需立即人工评估 | 是（强制优先审核） |
| CRITICAL | 3 | 紧急风险，需立即处理 | 是（强制优先审核） |

### 7.4 红旗标识

红旗标识代表需要立即关注的危险信号：

| 标识 | 说明 | 规则引擎风险等级 |
|------|------|:---:|
| consciousness_change | 意识状态改变 | CRITICAL |
| severe_breathing_difficulty | 严重呼吸困难 | CRITICAL |
| persistent_chest_discomfort | 持续性胸部不适 | HIGH |
| uncontrolled_bleeding | 无法控制的出血 | CRITICAL |
| self_harm_risk | 自伤风险 | CRITICAL |
| pregnancy_emergency_signal | 妊娠紧急信号 | HIGH |

## 9. 数据安全与隐私

### 9.1 PII 自动脱敏

系统在输入层自动检测并脱敏以下敏感信息：

| 类型 | 检测模式 | 脱敏替换 |
|------|----------|----------|
| 手机号 | 中国大陆 11 位手机号（1[3-9]开头） | [REDACTED_PHONE] |
| 身份证号 | 18 位身份证号 | [REDACTED_ID] |
| 电子邮箱 | 标准邮箱格式 | [REDACTED_EMAIL] |
| 银行卡号 | 16-19 位数字 | [REDACTED_BANK] |

### 9.2 审计日志记录

- 每次多 Agent 审核流程生成完整的 Agent 审计记录链
- 审计记录包含：Agent 名称、执行状态、输入摘要、输出摘要、开始时间、耗时、关联规则 ID
- 审计记录不包含：思维链、内部推理过程、原始敏感文本

### 9.3 合成数据专用

- 所有接口描述和响应中均包含安全声明
- 文档导入接口说明：内容必须为合成教学材料，不接受真实患者数据
- 评测数据集全部为合成生成
